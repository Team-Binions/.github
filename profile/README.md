### Beanions Rules

## 협업 rule
- Branch merge는 오전 수업 시작하기 전과 하원하기 전에 다같이 merge 한다.
- 개발 시작 전 이슈 작성
- merge시 완료된 이슈 닫기
- 개인 작업 내용은 각자 백업 해두기
- 개발 업무 일지 작성 (오전 회의 이후)


## 명명규칙
- issueTemplate제목[branchName][domainName]기능설명
- pullRequest제목[branchName][domain]기능설명
- branch 생성시 Name feature/구현기능명(camelCase)
